# ОТЧЕТ ПО ЛАБОРАТОРНОЙ РАБОТЕ №4

## Тема: Параллельное умножение матриц с использованием технологии CUDA

---

### 1. Цель работы

Модифицировать программу из лабораторной работы №1 для параллельной работы по технологии CUDA. Провести исследование эффективности параллельного умножения квадратных матриц на графическом процессоре (GPU) при различных размерах матриц и различных конфигурациях сетки блоков.

---

### 2. Ход выполнения работы

В ходе выполнения лабораторной работы была разработана программа на языке C++ с использованием технологии CUDA. Программа реализует умножение двух квадратных матриц на GPU. Для каждого размера матрицы тестировались пять различных конфигураций блоков: 16×16, 32×32, 64×64, 128×128 и 256×256 потоков на блок. Размеры матриц составили 200, 400, 800, 1200, 1600 и 2000 элементов. Для каждого эксперимента измерялось время выполнения и рассчитывалась производительность в MFLOPS.

### 5. Анализ результатов

#### 5.1. Зависимость производительности от размера блока

Для матрицы размером 200×200 наилучшая производительность (1 095 890 MFLOPS) достигнута при использовании блока 128×128. При использовании блока 32×32 производительность снизилась до 98 039 MFLOPS, что связано с избыточным количеством блоков для такого малого размера матрицы.

Для матрицы размером 400×400 наилучший результат (8 421 050 MFLOPS) также показал блок 128×128. При этом блок 16×16 показал результат 7 710 840 MFLOPS, что лишь незначительно уступает оптимальному значению.

Для матриц размером 800×800 и 1200×1200 наилучшие результаты показали блоки 16×16 и 32×32. Максимальная производительность для 800×800 составила 176 552 000 MFLOPS, для 1200×1200 - 500 870 000 MFLOPS.

Для больших матриц (1600×1600 и 2000×2000) блок 16×16 оказался наиболее эффективным. Производительность для 2000×2000 достигла 2 622 950 000 MFLOPS, что является максимальным значением среди всех экспериментов.

#### 5.2. Влияние размера матрицы на производительность

С увеличением размера матрицы производительность растет. Это объясняется тем, что на больших матрицах GPU может полностью загрузить все вычислительные ядра, минимизируя накладные расходы на запуск ядер и передачу данных. Для матрицы 200×200 производительность составляла около 1 млн MFLOPS, тогда как для матрицы 2000×2000 она достигла 2,6 млрд MFLOPS, что в 2600 раз выше.

### 6. Выводы

В ходе выполнения лабораторной работы была успешно разработана и протестирована программа для умножения квадратных матриц с использованием технологии CUDA. Проведены эксперименты на матрицах размером от 200×200 до 2000×2000 с конфигурациями блоков от 16×16 до 256×256.

Программа показала высокую эффективность при использовании GPU. Максимальная производительность 2 622 950 000 MFLOPS была достигнута на матрице 2000×2000 с блоком 16×16. Оптимальный размер блока зависит от размера матрицы: для небольших матриц лучше подходят блоки 128×128, для больших - блоки 16×16.

Установлено, что производительность GPU значительно превышает производительность CPU (в тысячи раз) при умножении матриц большого размера. CUDA позволяет эффективно загружать все вычислительные ресурсы GPU благодаря большому количеству параллельных потоков.

Результаты экспериментов показывают, что для достижения максимальной производительности необходимо правильно подбирать конфигурацию блоков и сетки в зависимости от размера решаемой задачи.
