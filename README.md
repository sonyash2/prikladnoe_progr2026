Отчет по лабораторной работе
Умножение матриц с параллельными вычислениями


1. Задание лабораторной работы

Разработать программу на языке C++ для перемножения двух квадратных матриц. Программа должна поддерживать как последовательный, так и параллельный режимы вычислений. Исходные данные могут загружаться из файлов или генерироваться случайным образом. Результаты должны сохраняться в файл. Обязательна верификация результатов с помощью стороннего ПО (Python/NumPy). Также необходимо провести исследование зависимости времени выполнения от размера задачи и параметров распараллеливания.

2. Теоретическая часть

Умножение двух квадратных матриц размера n×n требует выполнения n³ операций умножения и n³ операций сложения, то есть всего 2n³ арифметических операций. Вычислительная сложность алгоритма составляет O(n³).

Параллельное умножение матриц может быть реализовано путем разбиения матрицы на блоки строк. Каждый поток обрабатывает свой блок строк, что позволяет распределить вычислительную нагрузку между ядрами процессора. Согласно закону Амдала, максимальное ускорение ограничено долей последовательного кода в программе.

Для верификации результатов используется библиотека NumPy в Python, которая содержит оптимизированные функции для линейной алгебры и позволяет проверить корректность вычислений.

3. Реализация программы

Программа написана на языке C++ с использованием стандартной библиотеки. Для работы с потоками используется библиотека thread. Основные компоненты программы:

Класс MatrixMultiplier содержит все необходимые методы для работы с матрицами. В конструкторе происходит выделение памяти под матрицы A, B и C заданного размера. Метод generateRandomMatrices заполняет матрицы случайными числами от 1 до 10. Метод loadMatricesFromFile позволяет загрузить матрицы из текстового файла. Метод saveResultToFile сохраняет результат в файл.

Для последовательного умножения используется метод multiplySequential, который реализует классический алгоритм с тремя вложенными циклами. Для параллельного умножения метод multiplyParallel выполняет умножение для заданного диапазона строк. Метод runParallel создает потоки, распределяет между ними строки матрицы и запускает вычисления. Количество строк на поток вычисляется как n / num_threads, причем последний поток получает все оставшиеся строки.

Для верификации метод saveMatricesForVerification сохраняет матрицы A, B и C в файл verification_data.txt. Метод compareWithReference позволяет сравнить результат с эталонным файлом.

Программа работает в интерактивном режиме. Пользователь вводит размер матрицы, количество потоков, выбирает способ загрузки данных, сохранения результата и верификации. Время выполнения измеряется с помощью библиотеки chrono.

6. Анализ полученных результатов

Для матрицы небольшого размера (2×2) время выполнения составляет 0.005 секунд, что является ожидаемым результатом, так как выполняется всего 16 операций. Производительность 0.0032 MFLOPS кажется низкой, но это связано с тем, что для матриц малого размера накладные расходы на создание потоков и измерение времени сравнимы с временем самих вычислений.

Для получения более информативных результатов о масштабировании и эффективности параллельных вычислений необходимо проводить эксперименты на матрицах большего размера: 100×100, 200×200, 500×500, 1000×1000. Ожидается, что с ростом размера задачи:
1.Время выполнения будет расти пропорционально n³
2.Ускорение от параллельных вычислений будет приближаться к числу потоков для больших матриц
3.Эффективность будет снижаться при увеличении числа потоков из-за накладных расходов

8. Приложение. Листинг программы

Основные методы класса MatrixMultiplier:

Конструктор инициализирует матрицы заданного размера.
Метод generateRandomMatrices заполняет матрицы случайными числами.
Метод multiplySequential выполняет последовательное умножение.
Метод multiplyParallel выполняет умножение для блока строк.
